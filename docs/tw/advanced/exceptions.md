# 常見的程序異常

以下文檔列舉了 Snap Hutao 在使用過程中可能出現的常見問題及對應解決方案。

---

## **Return Code: 2002**

- **來源**: 米遊社返回的信息
- **問題原因**: 米哈遊論壇賬號未初始化
- **解決方案**:
  - 米遊社賬號：使用米遊社官方手機應用登錄賬號並設置社區用戶名以初始化。
  - HoYoLAB 賬號：使用 HoYoLAB 官方手機應用或網頁版登錄賬號並設置社區用戶名以初始化。

## **Return Code: -10001**

- **來源**: 米遊社返回的信息，包括 `Return Code: -10001` 和 `Return Code: -100`
- **問題原因**:
  1. Cookie 過期或網絡異常
  2. 系統時間錯誤
- **解決方案**:
  1. 在賬號面板中點擊“刷新 Cookie”。
  2. 刪除賬號後重新添加。
  3. 在系統設置中啟用自動同步時間並立即同步。

## **狀態：1034 驗證失敗**

- **問題原因**: 實時便箋功能被米遊社驗證系統阻擋。
- **解決方案**:
  1. 進入“實時便箋”功能。
  2. 點擊右上角“驗證當前用戶與角色”按鈕。
  3. 按提示完成驗證。
  4. 如果問題仍然存在，重複以上操作。

::: warning 注意事項

1. 修改密碼可重置設備登錄狀態。
2. 若綁定多個 Bot 或設備頻繁訪問米遊社數據，可能觸發米遊社的安全限制，暫時無法避免。
   :::

## **HttpRequestException 元數據校驗文件下載失敗**

### **ConnectionError**

- **原因**: 胡桃服務器連接失敗。
- **解決方案**: 檢查網絡代理或是否解除了 [Windows 容器 Loopback 限制](loopback.md)。

### **403 / 404 / 418**

- **解決方案**: 更新至[最新版本](../quick-start.md#全新安裝)。

### **502**

- **原因**: 胡桃服務器連接失敗。
- **解決方案**: 檢查網絡或聯繫開發團隊。

## **`653366069` 請求異常**

- **原因**: 無法連接至米遊社 / HoYoLAB 服務器。
- **解決方案**:
  - 忽略偶發錯誤。
  - 持續錯誤時檢查網絡連接。
  - 檢查是否解除了 [Windows 容器 Loopback 限制](loopback.md)。

## **資源下載失敗**

- 部分靜態資源下載失敗不會顯著影響使用，客戶端會在需要時自動重新嘗試。

### **`Received an unexpected EOF or 0 bytes from the transport stream.`**

- **原因**: 下載量限制或網絡波動。
- **解決方案**: 關閉胡桃稍後再試，或切換網絡環境。

## **在查找必要的模塊時出現問題**

- **錯誤信息**:
  1. 無法讀取模塊，可能是保護驅動加載問題。
  2. 查找模塊超時。
- **解決方案**: 胡桃會關閉遊戲進程，用戶再次嘗試啟動。

## **應用安裝失敗，錯誤消息：內部錯誤 0x80073D05**

- **原因**: 卸載後殘留文件未清理。
- **解決方案**:
  1. 打開 `%appdata%/../Local/Packages`。
  2. 刪除 `60568DGPStudio.SnapHutao_wbnnev551gwxy` 文件夾（權限不足時使用 `NSudo_LG`）。

## **提示 0x8007007E 或 找不到指定的文件/模塊**

- **原因**: 缺少 MSVC 運行庫。
- **解決方案**: 安裝最新版本 [MSVC 運行庫（x64）](https://aka.ms/vs/17/release/vc_redist.x64.exe)。
